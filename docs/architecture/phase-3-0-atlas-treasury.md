# Phase 3.0 Atlas Treasury Architecture

Date: 2026-03-21

## Overview

Atlas Treasury is now a real runtime-integrated sleeve foundation for idle-capital management. It is intentionally policy-driven and bounded:

- no allocator/meta-routing yet
- no fake live treasury connector claims
- deterministic dry-run-first behavior
- durable read models and operator visibility

## Package Boundaries

### `packages/treasury`

Owns:

- `TreasuryPolicy`
- `TreasuryReserveStatus`
- `TreasuryVenueSnapshot`
- `TreasuryRecommendation`
- `TreasuryEvaluation`
- `TreasuryPolicyEngine`

Responsibilities:

- compute reserve floor
- identify surplus idle capital
- detect concentration breaches
- detect ineligible venue exposure
- generate deterministic deposit/redeem recommendations

### `packages/venue-adapters`

Owns treasury venue abstractions:

- `TreasuryVenueAdapter`
- `TreasuryVenueState`
- `TreasuryVenuePosition`

Current implementation:

- `SimulatedTreasuryVenueAdapter`

### `packages/runtime`

Owns treasury orchestration:

- connects treasury adapters
- evaluates treasury during carry runtime cycles
- supports explicit `run_treasury_evaluation` runtime commands
- persists treasury runs, venue snapshots, actions, and current read models

## Runtime Flow

1. Carry cycle refreshes portfolio state and completes normal carry execution work.
2. The same refreshed portfolio state is passed into treasury evaluation.
3. Runtime reads treasury adapter state and current treasury positions.
4. `TreasuryPolicyEngine` computes:
   - reserve coverage
   - surplus capital
   - concentration posture
   - deposit/redeem recommendations
5. Runtime persists:
   - `treasury_runs`
   - `treasury_venue_snapshots`
   - `treasury_actions`
   - `treasury_current`
6. Runtime emits a treasury audit event for operator visibility.

## Persistence Model

### `treasury_runs`

One durable evaluation record with:

- source run linkage
- sleeve id
- simulated/live mode flag
- policy snapshot
- reserve metrics
- action count
- summary payload

### `treasury_venue_snapshots`

Per-venue allocation and liquidity snapshot for the treasury run.

### `treasury_actions`

Append-only recommended actions generated by the policy evaluation.

### `treasury_current`

Latest current treasury read model pointer and policy snapshot.

## Initial Policy Surface

- `GET /api/v1/treasury/policy`
- `GET /api/v1/treasury/actions`

Dashboard surfaces:

- overview treasury panel
- dedicated `/treasury` page
- safe operator action to queue treasury evaluation

## Honest Boundaries

- Treasury recommendations are persisted recommendations, not auto-executed capital transfers.
- Current venue support is simulated/dev only.
- Idle-capital input currently comes from the runtime portfolio tracker liquidity reserve rather than a full allocator.
- This foundation is intended to feed a later allocator phase, not replace it.
